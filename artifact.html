Artifact

School Project: Redesign | UI/UX

Context

Artifact is a personal podcast startup. Say you want to commemorate your sister's college graduation with a story detailing some of her best friends' favorite moments with her. Artifact is a hands-off personal-podcast making service that brings that story to fruition. They schedule the interviews, do the editing, and deliver the podcast in personalized website format.

For a school assignment, my group chose to make Artifact's website, which is where users place their orders. We focused on three things in particular: the order form, the order backend, and the product itself.

Feedback from Class Critique

  • “Don’t interfere in the relationship between the giver and the recipient”
  • Artifact makes personal gift, and so Artifact’s logo should be very minimal on the product page. Think of this as Digital Packaging. Think of the how product delivery should play out.
  • There is no need for the profile page to track Artifact received, that should be the responsibility of the receiver. Instead, only support gift management.
  • On “Your Orders”, the distinction between finished and unfinished Artifacts should be clearer.
  • “Call to Action” ---> Move the ‘Get Started’ button above the fold

User Testing

Lastly, we did some user testing to see how potential users would interact with our. We gave three users a set of tasks and asked them some questions.

Tasks

Task 1

Order an artifact.

Task 2

Test general navigation of our prototype by completing the following subtasks:

  • Navigate to the account/profile page.
  • Search for an order, then return to the account/profile page.
  • Figure out the state of your most recent order.
  • Open a completed Artifact.

Questions

  1. Which task was the most confusing? What did you expect?
  2. Which task was the most enjoyable? Why was it enjoyable?

Test Analysis

User 1

User 1 was able to easily order the artifact, using the term “simple”, “straightforward”, “easy”, “quick”. He was then able to navigate to the profile page, search for an order, figure out the state of the most recent order, and open a completed Artifact. This user’s actions were spot on with the tasks that we provided and in line with our expectations of the user testing.

User 2

User 2 took his time to go through the website/reading packages before working on his tasks, leaving comments such as “pretty cool layout”. He was able to order an artifact, calling the process “straight to the point”, “simple”, and “awesome”. He thought there could’ve been more interview styles to choose from, which is something we can consider when redesigning the website. He was able to navigate to the profile page, figure out the state of the most recent order, and open a completed Artifact without a problem.

User 3

It was relatively very quick for User 3 to order an artifact, and then navigate to the profile page, search for an order, figure out the state of the most recent order, and open a completed Artifact. He left comments such as “nice”, “very easy to navigate”, “very easy to order”.

Overall

Overall there were no certain types of errors the users made, and they all completed the required tasks. One potential interface change could be adding more details to the Home page, as User 2 has commented it “could be too simple”.

Takeaways

I had an excellent time working with my two other group members, and it was great fun to conceptualize a design for an emerging startup. I learned how to plan across timezones (my partners were both in Thailand), take feedback from others, iterate, and improve a design.
